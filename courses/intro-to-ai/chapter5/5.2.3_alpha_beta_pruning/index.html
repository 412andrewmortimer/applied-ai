    üìñ 5.2.3 Alpha‚ÄìBeta Pruning¶

    üìå Introduction¶

    In the previous sections, we explored the Minimax Algorithm for making optimal decisions in two-player zero-sum games. However, Minimax has a major weakness: it evaluates all possible moves, which becomes computationally expensive for large games like Chess or Go.

    Alpha-Beta Pruning is an optimization technique for Minimax that significantly reduces the number of nodes that need to be evaluated, making Minimax faster without affecting the final decision.

    ‚úÖ Why Use Alpha-Beta Pruning?¶

    ‚úî Reduces the number of nodes that must be explored.
    ‚úî Maintains the same optimal decision as Minimax.
    ‚úî Speeds up decision-making in complex games like Chess.
    ‚úî Efficiently searches deeper levels in the game tree.

    üîπ What is Alpha-Beta Pruning?¶

    Alpha-Beta Pruning eliminates branches in the game tree that won‚Äôt influence the final decision. It uses two values: - Alpha (Œ±): The best value MAX can guarantee so far. - Beta (Œ≤): The best value MIN can guarantee so far.

    During the Minimax search: - If MIN finds a move worse than an already evaluated option, it stops searching that branch.
    - If MAX finds a move better than an already evaluated option, it stops searching that branch.

    ‚úÖ This saves time by avoiding unnecessary calculations.

    üîπ Alpha-Beta Pruning Algorithm Steps¶

    1. Set Œ± = -‚àû (worst for MAX) and Œ≤ = +‚àû (worst for MIN).
    2. Traverse the game tree using Minimax.
    3. At each node:
    4. If MAX finds a move better than Œ≤, stop evaluating that branch (prune).
    5. If MIN finds a move worse than Œ±, stop evaluating that branch (prune).
    6. Continue the search while updating Œ± and Œ≤ values dynamically.
    7. Return the best possible move for MAX.

    üîπ Alpha-Beta Pruning Pseudocode¶

    Here is a recursive implementation of Alpha-Beta Pruning:

    def alphabeta(state, depth, alpha, beta, is_maximizing):
    if is_terminal(state) or depth == 0:
        return evaluate(state)

    if is_maximizing:
        best_value = -float("inf")
        for move in get_possible_moves(state):
            new_state = apply_move(state, move)
            value = alphabeta(new_state, depth - 1, alpha, beta, False)
            best_value = max(best_value, value)
            alpha = max(alpha, best_value)  # Update Œ±

            if beta <= alpha:  # Beta Cutoff (Prune)
                break
        return best_value

    else:  # Minimizing Player (MIN)
        best_value = float("inf")
        for move in get_possible_moves(state):
            new_state = apply_move(state, move)
            value = alphabeta(new_state, depth - 1, alpha, beta, True)
            best_value = min(best_value, value)
            beta = min(beta, best_value)  # Update Œ≤

            if beta <= alpha:  # Alpha Cutoff (Prune)
                break
        return best_value

    ‚úÖ What this does:
    - Updates Œ± (MAX‚Äôs best move) and Œ≤ (MIN‚Äôs best move) dynamically.
    - Prunes branches when Œ≤ ‚â§ Œ± (no need to explore further).
    - Ensures the same optimal move as Minimax but faster.

    üîπ Time Complexity of Alpha-Beta Pruning¶

    Algorithm Time Complexity Explored Nodes
    Minimax O(b^d) Explores all nodes
    Alpha-Beta Pruning O(b^(d/2)) Explores fewer nodes

    ‚úÖ Why is Alpha-Beta Pruning Faster? - Pruning eliminates branches that won‚Äôt influence the final decision. - Allows the AI to search deeper in the same amount of time.

    üîπ Strengths & Weaknesses of Alpha-Beta Pruning¶

    ‚úÖ Advantages¶

    ‚úî Significantly reduces computation time compared to Minimax.
    ‚úî Allows deeper searches in complex games like Chess.
    ‚úî Still finds the same optimal move as Minimax.

    ‚ùå Disadvantages¶

    ‚úñ Does not work well if moves are unordered (bad ordering reduces pruning).
    ‚úñ Still exponential complexity (O(b^(d/2)), but better than O(b^d)).
    ‚úñ Limited usefulness in non-zero-sum games (e.g., Poker).

    üîπ Key Takeaways¶

    ‚úî Alpha-Beta Pruning speeds up Minimax without changing the final decision.
    ‚úî Pruning occurs when we find a move that makes further search unnecessary.
    ‚úî The best case occurs when moves are sorted from best to worst.
    ‚úî Allows AI to play games like Chess and Go more efficiently.
